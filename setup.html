Before you start

What you'll need

Three free things. Install the first two once and you're set for good.

Free

Node.js 20

The engine Harmonix runs on. Download the LTS version from nodejs.org and click through the installer.

Free

Docker Desktop

Runs the music engine for you — the launcher starts it automatically. Free from docker.com. (Prefer no Docker? You can run the engine with Java 17+ instead — see step 5.)

Free

A Discord bot

Made in a couple of clicks at the Developer Portal. Step 1 walks you through it.

1

Create your Discord bot

This is the account your bot logs in as. It's free and takes about two minutes.

Open the Developer Portal, click New Application, and name it Harmonix. Then grab two things:

  • On the Bot tab, click Reset Token and copy the token. This is the bot's password — you'll only see it once, so paste it somewhere safe.
  • Still on the Bot tab, scroll to Privileged Gateway Intents and switch on Server Members and Message Content.
  • On the General Information tab, copy the Application ID (also called the Client ID). You'll need it in a moment.

Keep the token private. Anyone who has it can control your bot. If it ever leaks, hit Reset Token again and the old one stops working instantly.

2

Install Harmonix

Unpack the download and let it grab what it needs.

Unzip the Harmonix folder wherever you like. Open a terminal inside that folder and run:

terminal
npm install

This pulls in everything the bot and dashboard need. The music engine comes with the project too — the launcher spins it up for you the first time you start, so there's nothing else to install by hand.

Not sure how to open a terminal in a folder? On Windows, open the folder, click the address bar, type cmd and press Enter. On Mac, right-click the folder → Services → New Terminal at Folder.

3

Basic setup

Tell Harmonix which bot it is. Just two values to fill in.

In the folder you'll find a file called .env.example. Make a copy and rename the copy to .env, then open it in any text editor and paste in your token and ID from step 1:

.env
DISCORD_TOKEN=paste-your-bot-token-here
CLIENT_ID=paste-your-application-id-here

Everything else in that file already has a sensible default. Save it and move on.

Want your slash commands to appear instantly while testing? Paste your server's ID into GUILD_ID. Leave it blank and commands still work — they just take up to an hour to show up the first time.

4

Pick how you log into the dashboard

Two ways in — both completely free. Most people want the first one.

Easiest · recommended

🔒 Local password

You open the dashboard on the same computer that runs the bot.
  • No domain, no tunnel, nothing to sign up for
  • Just set one password
  • Works the moment you launch

🎮 Discord login

You'd rather sign in with your Discord account.
  • Still free, still runs locally
  • Only lets people manage servers they already run
  • A couple of extra fields to fill

For the local password, add one line to your .env and you're done:

.env
# Leave CLIENT_SECRET blank to stay in local mode,
# then set a password to open the dashboard with:
DASHBOARD_PASSWORD=pick-any-password

Prefer the Discord login? On the Developer Portal's OAuth2 tab copy your Client Secret, then fill these in. Leave the redirect line blank and Harmonix points it at your own computer automatically:

.env
CLIENT_SECRET=your-client-secret
SESSION_SECRET=any-long-random-string
# Leave OAUTH_REDIRECT_URI blank — it auto-fills to
# http://localhost:3000/auth/callback

Then, on that same OAuth2 tab, add http://localhost:3000/auth/callback under Redirects so Discord recognises it.

5

Launch — just double-click

Once everything's installed, you don't touch the terminal at all. Harmonix ships with a launcher that does the whole start-up for you.

On Windows, open the Harmonix folder and double-click start.bat. On Mac or Linux, run ./start.sh. That one file:

  • Makes sure the music engine is up (starts it through Docker for you)
  • Registers the slash commands so they show up in Discord
  • Opens a free public link for the dashboard and prints the exact login redirect to paste into Discord
  • Starts the bot and streams everything into one window

That's the whole job. Install the pieces once, double-click the launcher, and Harmonix runs. Leave that window open while you're using the bot; closing it (or Ctrl+C) stops the bot.

Want it out of the way? On Windows, double-click start-silent.vbs instead and Harmonix runs hidden with an icon in your system tray — right-click it to open the dashboard, restart, or stop.

Rather not use the launcher? You can start things by hand — the launcher just runs these for you:

terminal
# the same thing the batch file does
npm run launch

# local only — skip the public link, dashboard stays on your PC
npm run launch:local

# no Docker? run the music engine yourself with Java 17+ first…
cd lavalink && java -jar Lavalink.jar
# …then in another window:
node run.js --no-tunnel

However you start it, open http://localhost:3000 in your browser and log in the way you chose in step 4. That's the dashboard.

6

Invite the bot to your server

One link adds it. Commands set themselves up on their own.

Paste your Client ID into this link where it says YOUR_CLIENT_ID, open it, pick your server, and confirm:

invite link
https://discord.com/oauth2/authorize?client_id=YOUR_CLIENT_ID&permissions=277083450646&scope=bot+applications.commands

Slash commands deploy automatically on first launch. If you left GUILD_ID blank, give them a few minutes to appear the first time.

Optional · also free

Open the dashboard from another device

Everything above runs on your own machine, which is all most people need. If you also want to reach the dashboard from your phone or another computer, you can — without paying for a domain — using a free Cloudflare quick tunnel:

terminal (second window)
cloudflared tunnel --url http://localhost:3000

It prints a free https://…trycloudflare.com address you can open anywhere. Put that address in PUBLIC_URL in your .env, and if you use the Discord login, add <that address>/auth/callback to your OAuth2 redirects too.

No port forwarding, ever. You don't open any router ports for this. The tunnel makes an outbound connection to Cloudflare, so it works from behind any home network with nothing to configure — that's the whole reason to use it instead of forwarding a port.

Heads up: a free quick-tunnel address changes every time you restart it, so you'd update those two spots each run. Fine for occasional remote access — for the local password setup you never need this at all.

If something's off

The usual hiccups, and the one-line fix for each.

Do I need to port-forward or open router ports? +
No. The bot connects out to Discord, so it receives commands without any port being opened — it just works behind your router. The music engine runs locally on your PC. The only thing that ever leaves your machine is the optional dashboard tunnel, and that dials out to Cloudflare, so there's still nothing to forward. If a guide tells you to open port 3000 on your router, you don't need to.
The dashboard login won't work +
If you're using the local password, double-check CLIENT_SECRET is blank and DASHBOARD_PASSWORD is set. If you're using the Discord login and it keeps bouncing you back, the cause is almost always a leftover OAUTH_REDIRECT_URI line — delete it so Harmonix can fill it in automatically, and make sure the same address is listed under Redirects in the Developer Portal.
Music won't play +
This is the music engine (Lavalink). Make sure Docker Desktop is running before you launch — the launcher starts the engine through it. Check the startup logs actually show the engine connecting. (Running it the no-Docker way instead? Confirm Java 17+ with java -version and that Lavalink.jar is up.)
Slash commands aren't showing up +
Global commands can take up to an hour the very first time. To see them instantly while setting up, paste your server's ID into GUILD_ID and relaunch. Also make sure you invited the bot with the link in step 6, which includes the commands permission.
It logs me out every time I restart +
Set SESSION_SECRET to any long random string in your .env. When it's blank, a new one is generated on each restart, which signs everyone out.
The bot is online but ignores commands +
Usually a missing intent. Go back to step 1 and make sure both Message Content and Server Members are switched on in the Developer Portal, then restart.
